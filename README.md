# 使用git 版本控制总结

## 版本控制系统的优势

### 1.获得每个文件的长期更改历史。

### 2. 分支和合并（branch & merge）

#### 这样两个功能使得团队成员同时工作

### 3. 可追溯的（Traceability）

#### 能够跟踪对软件所做的每一项修改，并将其与项目管理软件和错误跟踪软件连接在一起

#### 能够注释每次更改
